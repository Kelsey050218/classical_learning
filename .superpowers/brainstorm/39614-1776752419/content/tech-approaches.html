技术方案对比 - 3 种架构方案

基于 React + FastAPI + MySQL + 阿里云 OSS 的技术栈

方案 A：前后端分离 + RESTful API（推荐）

🖥️ 前端（React）

  • 框架：Vite + React + TypeScript
  • 路由：React Router v6
  • 状态管理：Zustand（轻量）
  • UI 库：Ant Design / TailwindCSS
  • HTTP：Axios + 拦截器（JWT 自动注入）
  • 富文本：TipTap / Slate（批注/笔记）

⚙️ 后端（FastAPI）

  • 框架：FastAPI + Uvicorn
  • ORM：SQLAlchemy + Alembic
  • 认证：JWT (access + refresh token)
  • 存储：阿里云 OSS SDK
  • 密码加密：Bcrypt (passlib)
  • 部署：Gunicorn + Uvicorn workers

📐 数据流

浏览器 (React SPA)
↔ HTTP/REST
Nginx 反向代理 → FastAPI
↔ SQL
MySQL
FastAPI
↔ OSS SDK
阿里云 OSS
✅ 优势：前后端独立开发、独立部署；React 生态丰富；FastAPI 性能好；RESTful 清晰
❌ 劣势：CORS 配置、JWT 管理需要额外代码
⏱ 预估工期：8-10 个工作日
方案 B：Next.js + FastAPI（SSR 方案）

🖥️ 前端（Next.js）

  • 框架：Next.js (App Router)
  • 渲染：SSR + SSG 混合
  • 状态：React Server Components
  • 优势：SEO 友好、首屏快

⚙️ 后端（FastAPI）

  • 与方案 A 相同
  • 提供 RESTful API
  • JWT 认证
  • OSS 文件管理
✅ 优势：首屏加载快、SEO 好
❌ 劣势：Next.js 学习曲线陡；交互组件需要 client 端 hydration；对伴学工具来说 SEO 不是刚需
⏱ 预估工期：10-12 个工作日
方案 C：React + Django REST Framework

🖥️ 前端（React）

  • 与方案 A 相同
  • CSR 客户端渲染

⚙️ 后端（Django REST）

  • 框架：Django REST Framework
  • ORM：Django ORM
  • 优势：内置 Admin、内置认证
  • 劣势：重量级、启动慢
✅ 优势：Django Admin 开箱即用（内容管理后台）、生态最成熟
❌ 劣势：Django 重量级、性能不如 FastAPI、异步支持弱
⏱ 预估工期：8-10 个工作日

🏆 推荐：方案 A（React SPA + FastAPI）
理由：你熟悉 React 和 FastAPI；CSR 对伴学工具足够；FastAPI 轻量高性能；10 天工期可控。
